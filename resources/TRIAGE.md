# Triage Guidelines

This document defines how maintainers triage issues and pull requests.

## Labels

Recommended baseline labels:

- `triage`: Needs maintainer classification
- `bug`: Something is broken
- `enhancement`: Process or documentation improvement
- `good first issue`: Suitable for first-time contributors
- `help wanted`: Maintainers welcome community help
- `documentation`: Documentation-focused work

## Triage Workflow

1. Confirm the issue type and apply labels.
2. Link related docs or existing issues to avoid duplication.
3. Assign owner or request maintainer review.
4. Mark `good first issue` only when scope is clear and self-contained.
5. Close stale or invalid issues with a clear reason.

## Pull Request Triage

- Ensure template checklist is completed.
- Confirm contribution is in the correct directory.
- Verify links and markdown formatting.
- Request changes with actionable feedback when needed.
